# Development Guidelines for azure-to-drawio

## Python Environment

**IMPORTANT: Always use the `.venv` virtual environment for all Python operations.**

- The project includes a `.venv` directory at the repository root
- **Activate the environment before running any Python commands:**
  ```bash
  source .venv/bin/activate
  ```
- All pytest commands, pip installs, and Python script execution must happen within the activated `.venv`
- This applies to both terminal commands and subprocess operations

### Running Tests
```bash
source .venv/bin/activate
python -m pytest tools/azdisc/tests/ -x -q
```

### Running the Application
```bash
source .venv/bin/activate
cd tools/azdisc_ui
python -m azdisc_ui
```

## Project Structure

- `tools/azdisc/`: Core discovery and diagram generation engine
- `tools/azdisc_ui/`: FastAPI web UI for the discovery tool
- `tools/azdisc/tests/`: Test suite (pytest)
- `assets/`: Icon maps and draw.io libraries
- `skills/`: Custom agent skills documentation

## Key Implementation Notes

### Diagram Modes
The tool supports three diagram rendering modes (via `cfg.diagramMode`):
1. **MSFT**: Microsoft Architecture Center style with region/RG/type hierarchies
2. **L2R**: Left-to-right traffic flow layout with resource dependency chains
3. **HUB-SPOKE**: Hub-spoke network topology with subscription swimlanes and enterprise coloring (for SAP/complex environments)

Mode selection is wired through:
- Config validation: `tools/azdisc/config.py` - `VALID_DIAGRAM_MODES = {"MSFT", "L2R", "HUB-SPOKE"}`
- Rendering dispatch: `tools/azdisc/drawio.py` - `generate_drawio()` function routes to `_render_msft_mode()`, `_render_l2r_mode()`, or `_render_hub_spoke_mode()`
- UI controls: `tools/azdisc_ui/templates/index.html` - select elements with mode options
- Frontend state: `tools/azdisc_ui/static/app.js` - passes `diagramMode` to backend API endpoints

### Backend API Endpoints

All diagram generation endpoints accept the following style parameters:
- `diagramMode`: "MSFT" | "L2R" | "HUB-SPOKE"
- `spacingPreset`: "compact" | "spacious" (or numeric 0-100 range mapped to spacing factor)
- `layoutMagic`: boolean (enable intelligent node clustering)
- `edgeLabels`: boolean (show/hide relationship labels)
- `subnetColors`: boolean (apply tier-based subnet coloring)

Key endpoints:
- `POST /api/diagram/generate-scoped` - generates diagram from selected scope
- `POST /api/diagram/generate-selection` - generates diagram from resource selection
- `POST /api/artifacts/export-bundle` - bundles multiple diagram artifacts into a zip
- `GET /api/diagram-beta/viewer` - serves local draw.io viewer (vendor/viewer-static.min.js)

### Testing

Run the full test suite:
```bash
source .venv/bin/activate
python -m pytest tools/azdisc/tests/ -x -q
```

Individual test files can be run to isolate issues:
```bash
python -m pytest tools/azdisc/tests/test_drawio_schema.py -v
python -m pytest tools/azdisc/tests/test_integration.py -v
```
